## v1.0：单机多线程，打通 TCP 通信

### 目标
- 客户端和服务端能互相收发消息。
- 支持多个客户端同时连接（每个连接一个线程）。

### 需要掌握的知识点
| 知识点 | 具体内容 | 学习资源建议 |
| :--- | :--- | :--- |
| **Socket API** | `socket()`, `bind()`, `listen()`, `accept()`, `connect()`, `send()`, `recv()` | 《Unix 网络编程》第1-4章 |
| **TCP 协议基础** | 三次握手、四次挥手、状态转换图 | 网上搜“TCP 状态机” |
| **多线程基础** | `std::thread` 创建与 detach/join，`std::mutex` 保护共享数据 | 《C++ Primer》第12章 |
| **简单应用层协议** | 如何定义消息边界（定长、分隔符、头部声明长度） | 自己设计一个简单的 `|` 分隔协议 |

### 交付物
- [ ] `server.cpp`：能接受多个客户端连接，并回显消息。
- [ ] `client.cpp`：能连接服务端，发送消息，接收回显。
- [ ] 代码能在一台 Linux 虚拟机上跑通。

## v2.0：Epoll + 线程池，高并发骨架

### 目标
- 用 Epoll 替代多线程阻塞 I/O，用线程池处理业务逻辑。
- 支持 1000+ 并发连接。

### 需要掌握的知识点
| 知识点 | 具体内容 | 学习资源建议 |
| :--- | :--- | :--- |
| **Epoll** | `epoll_create`, `epoll_ctl`, `epoll_wait`，LT 与 ET 模式区别 | 《Unix 网络编程》第6章，或搜“Epoll 边缘触发” |
| **非阻塞 I/O** | `fcntl` 设置 `O_NONBLOCK`，处理 `EAGAIN` 错误 | 同上 |
| **Reactor 模式** | 主线程负责 I/O 事件分发，工作线程负责业务处理 | 搜“Reactor 模式 C++” |
| **线程池** | 任务队列 + 多个工作线程，`std::condition_variable` 实现生产者-消费者 | 自己手写一个简单版 |
| **MySQL 接入** | 使用 MySQL C API 或 `mysqlclient` 库，实现用户注册/登录 | 搜“MySQL C++ 连接示例” |

### 交付物
- [ ] `Epoller` 类封装 Epoll 操作。
- [ ] `ThreadPool` 类实现任务提交与执行。
- [ ] `Connection` 类管理每个客户端的状态和缓冲区。
- [ ] 用户数据存入 MySQL，支持注册和登录验证。
- [ ] 用压测工具（如 `wrk` 或自写客户端）测出 1000 并发下的 QPS。

## v2.5：工程化完善，工业级单机版

### 目标
- 让 v2.0 的项目具备工业级软件的健壮性和可维护性。

### 需要掌握的知识点
| 知识点 | 具体内容 | 学习资源建议 |
| :--- | :--- | :--- |
| **日志系统** | 用 `spdlog` 或自己封装，支持分级、文件滚动、异步日志 | `spdlog` 官方文档 |
| **配置管理** | 从 `json` 或 `ini` 文件读取配置，支持命令行参数覆盖 | 用 `nlohmann/json` 库 |
| **单元测试** | 用 `Google Test` 为线程池、协议解析等模块写测试 | `Google Test` 入门教程 |
| **协议增强** | 设计二进制协议（头部定长 + 变长 Body），解决粘包 | 自己设计并实现 `Codec` 类 |
| **连接管理** | 实现心跳机制，踢出死连接 | 定时器 + 超时检测 |
| **离线消息** | 存入 MySQL，用户上线后拉取 | SQL 设计与分页查询 |

### 交付物
- [ ] 完整的日志输出，关键路径均有记录。
- [ ] 所有配置项从文件读取，无硬编码。
- [ ] 核心模块的单元测试覆盖率 > 60%。
- [ ] 稳定的二进制协议，支持粘包处理。
- [ ] 心跳与离线消息功能完整。

## v3.0：分布式微服务架构

### 目标
- 将单体服务拆分为多个微服务，支持横向扩展。
- 引入工业级分布式组件，实现服务发现、RPC 通信和消息队列

### 需要掌握的知识点
| 知识点 | 具体内容 | 学习资源建议 |
| :--- | :--- | :--- |
| **gRPC** | 定义 `.proto` 文件，生成 C++ 服务端/客户端代码 | gRPC 官方教程 |
| **Protobuf** | 消息序列化，`repeated`、`map` 等复合类型 | Protobuf 官方文档 |
| **etcd** | 服务注册与发现，租约与心跳 | etcd C++ 客户端使用 |
| **Nginx** | TCP 负载均衡配置 | Nginx `stream` 模块 |
| **RabbitMQ / Kafka** | 消息队列的基本使用，解耦服务间通信 | 选一个，跑通 Hello World |

### 交付物
- [ ] 拆分出至少 **网关服务(Gateway)** 和 **逻辑服务(Logic)**。
- [ ] 服务间通信使用 **gRPC + Protobuf**。
- [ ] 网关能通过 **etcd** 自动发现逻辑服务的地址。
- [ ] 引入 **RabbitMQ** 或 **Kafka** 处理跨服务异步消息。
- [ ] 支持多网关、多逻辑服务的集群部署。
- [ ] 提供一张手绘或工具绘制的 **系统架构图**

## v4.0：极致性能与高可用

### 目标
- 对系统核心路径进行深度性能优化，掌握底层调优技术。
- 构建高可用的容灾机制。

### 交付物
- [ ] 实现 **内存池** 替代频繁的 `new/delete`，降低内存碎片。
- [ ] 在网关层实现 **零拷贝** 转发，提升吞吐量。
- [ ] 将线程池改造为 **无锁队列** 或 **lock-free 结构**。
- [ ] 实现服务间的 **灰度发布** 与 **蓝绿部署** 策略。
- [ ] 完成一份 **性能压测与调优报告**，包含优化前后的数据对比。
