# 分布式高并发聊天系统

## 项目目标
为中小公司C++后端岗位准备的工业级项目。证明具备独立开发、调试、优化高并发分布式系统的能力。

## 版本演进路线图

| 版本 | 核心目标 | 预估代码量（含测试） |
| :--- | :--- | :--- |
| v1.0 | 单机多线程，打通TCP通信 | 500 - 800 行 |
| v2.0 | Epoll + 线程池，高并发骨架 | 1,500 - 2,500 行 |
| v2.5 | 工程化完善（日志/配置/单测/协议/离线消息） | 8,000 - 12,000 行 |
| v3.0 | 分布式微服务（gRPC/etcd/MQ） | 12,000 - 18,000 行 |
| v3.5 | 分布式工程化（监控/链路追踪/容器化） | 18,000 - 25,000 行 |
| v4.0 | 极致性能与高可用（内存池/零拷贝/熔断） | 25,000 - 35,000 行 |
